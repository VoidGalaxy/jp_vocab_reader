# 시오리(Shiori) 캐릭터 디자인 스펙

> 기준 문서: `docs/design/reference/시오리 캐릭터 디자인 시안.png`
> 이 문서는 위 참고 이미지를 최우선 기준(reference of truth)으로 삼아 정리한 브랜드 캐릭터 스펙입니다.
> 구현 시 이 문서를 기준으로 하고, 참고 이미지를 픽셀 단위로 그대로 베끼지 않습니다 (외부 이미지 사용 금지, React/SVG/CSS로 재해석 구현).

## 핵심 정체성 요약

시오리는 "귀여운 코지 마스코트에 책갈피 장식을 붙인 것"이 아니라, **몸 자체가 책갈피인 존재**다. 세로로 긴 실루엣, 뚜렷한 V컷, 옆으로 드리운 고리·태슬·잎사귀 참이 장식이 아니라 캐릭터 정체성의 본체다. 표정은 항상 평온·졸림 쪽에 머물고, variant는 색이나 소품 하나를 바꾸는 수준이 아니라 몸의 자세 자체가 상황마다 달라지는 방식으로 구분된다. "책과 함께 있을 때 가장 자연스러운 캐릭터"라는 관계성이 최우선이다.

---

## 1) 디자인 원칙

| 항목 | 내용 |
|---|---|
| 캐릭터 컨셉 한 줄 | 책 속에 살며 조용히 곁을 지키는 코지한 북마크 정령 — 브랜드 캐릭터이자 UI 기능 일러스트를 동시에 만족 |
| 외형 핵심 | 세로로 긴 북마크 실루엣(폭:높이 ≈ 1:2 이상), 둥근 상단 + 아래쪽 뚜렷한 V컷(리본 컷, 실루엣의 25~30%), 짧고 말랑한 팔다리 |
| 얼굴/표정 규칙 | 기본은 평온·졸림. 표정 종류는 평온/졸림/미소(+아주 미세한 놀람, 필요시)를 넘지 않음. 볼터치는 모든 표정 공통. 눈매·입꼬리 각도만 미세 조정, 과장된 만화 표정 금지 |
| 책갈피 요소 규칙 | ① 고리(중앙 아닌 옆 위치) ② 태슬/끈(반드시 옆으로 드리워짐 — 수직이면 안테나) ③ 잎사귀 참(태슬 끝, 몸통과 겹침) ④ 아래쪽 V컷(뚜렷하고 깊게). 4가지 모두 모든 variant·모든 크기에서 생략 불가 |
| 컬러/라인 규칙 | 바디: 크림/아이보리 고정. 라인: 짙은 브라운 계열 고정(틸 아님). 포인트: 고리(amber/gold) · 참(sage green) · 볼터치(peach) 3개로 제한. variant별 몸 색 변경 금지 |
| 금지사항 | 마시멜로/떡/유령형 원통 실루엣 · 태슬이 수직 안테나처럼 보이는 것 · 팔다리가 얇은 실선(철사) · 과장된 표정·다색 장식 · variant별 몸통 색/실루엣 변경 · sm 크기에서 뭉개지는 디테일 |

## 2) 최종 방향

> 세로로 긴 북마크 실루엣 + 한쪽으로 드리운 고리·태슬·잎사귀 참 + 짧고 말랑한 팔다리 + 항상 평온/졸린 표정을 가진, 책과 함께 있을 때 가장 자연스러운 코지 북마크 정령.

강화 포인트: V컷을 더 깊고 뚜렷하게 / 팔다리를 몸의 일부처럼 두껍게, variant마다 각도·무게중심 다르게 / 태슬 드리움 각도를 확실히 옆으로(30~45도 이상).

## 3) Variant 설계 (9종)

| Variant | 상황 | 포즈/소품 | 표정 | 책갈피 요소 | 작은 크기 핵심 |
|---|---|---|---|---|---|
| default | 기본 마크/스탬프 | 정면, 소품 없음 | 평온 | 고리+태슬+참+V컷 상시 노출 | 실루엣+고리/태슬 흔적 |
| hero | 홈 대표 일러스트 | 책 위에 기대어 휴식, 팔 접음, 다리 늘어뜨림 | 졸림 | 몸이 책 위에 얹힌 구도, 태슬이 책 옆으로 늘어짐 | 책 소품 유지(항상 큰 사이즈) |
| reading | 읽기 안내 | 작은 책을 들고 보는 자세 | 평온(눈 살짝 뜸) | 손의 책이 정체성과 연결 | 고개 숙인 실루엣만으로 인식 |
| classify | 빠른 분류 | 단어 카드 한 장을 들어 보임 | 평온~미소 | 카드를 책에 끼우는 제스처 | 카드 하나만 남겨도 인식 |
| save | 저장 완료 | 카드함(노트)에 단어를 넣음 | 평온, 만족 | 카드함=책갈피 자리 은유 | 숙인 자세만으로 인식 |
| review | 복습 대기 | 작은 책을 품에 안음 | 졸림 | 안은 책이 자신과 같은 톤 | 안은 팔 실루엣만으로 인식 |
| success | 완료 스탬프 | 양팔 살짝 들고 반짝임 | 미소(가장 밝은 표정) | 태슬/참이 함께 흔들림 | 반짝임+미소만으로 인식 |
| empty | 빈 카드함/덱/노트 | 빈 상자 옆에 서거나 기댐 | 평온~살짝 처짐 | 상자 옆에서도 고리/태슬 유지 | 상자 없이도 "기다림" 인식 |
| loading | 로딩/분석 중 | 책더미에 기대 꾸벅꾸벅, 작은 Zzz | 완전히 감은 눈, 고개 기울임 | 책더미와 겹쳐 "책 사이" 인상 | 고개 각도만으로 인식, Zzz 생략 가능 |

## 4) UI 적용 가이드

- 홈 hero: `hero` variant, 큰 사이즈, 책 일러스트 위에 겹침. CTA보다 조용하게.
- 읽기: `reading` variant, 안내 카드 내부 통합(플로팅 금지), sm~md.
- 빠른 분류: `classify` variant, 인트로 md~lg, CTA가 가장 눈에 띄어야 함.
- 복습: ready=`review`, 카드 없음=`empty`, 완료=`success`. 항상 rating 버튼보다 작게.
- 노트: `empty`만, row 반복/상단 hero 금지.
- 덱/통계: `empty`/`loading`/`success`만 아주 작게, 숫자·덱 카드보다 튀지 않게.

## 5) 구현 명세 (SVG/벡터)

```
- viewBox 비율: 세로:가로 ≈ 1.4:1
- Body: 둥근 상단 + 뚜렷한 V컷(실루엣의 25~30%)
- Ring: 몸통 중심에서 옆으로 오프셋 (amber/gold)
- Tassel string: curve 곡선으로 "옆으로 흐름" — 수직 금지
- Charm(leaf): string 끝, 몸통과 겹침 (sage green)
- Arms/Legs: thick rounded stroke, variant마다 각도/무게중심 다르게
- Face: eyes(닫힘/반쯤뜸/뜸), blush(모든 variant 공통), mouth(단순 곡선)
- Props: variant당 1개 이하, 몸통과 동일 라인 두께
- Size steps: sm/md/lg/xl/hero — 모든 단계에서 세로 실루엣·고리+태슬·눈·V컷·참 유지
- Color tokens: body(cream)/line(brown)/ring(amber)/charm(sage)/blush(peach)
- Motion: loading만 미세 tilt, prefers-reduced-motion 대응
- 외부 이미지 사용 금지 — React/SVG/CSS만
```
