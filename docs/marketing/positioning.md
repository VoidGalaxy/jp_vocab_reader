# 포지셔닝 문서 — 책갈피 (일본어 원문 읽기 단어장)

> 이 문서는 랜딩/서비스 카피, SEO metadata, 온보딩 문구를 쓰거나 고칠 때 참고하는 기준 문서입니다.
> 디자인 톤은 `docs/design/ui-guidelines.md`, 캐릭터 사용은 `docs/design/shiori-design-spec.md`, 저장 정책의 기술적 배경은 `docs/architecture/shared-lexeme-progress-storage.md`를 따릅니다. 이 문서는 그 위에 "무슨 말을 할지"만 다룹니다.

## 1. 서비스 한 줄 설명

**일본어 원문·웹소설을 읽으면서 모르는 단어만 저장하고, SRS로 복습하는 일본어 단어장 서비스입니다.**

앱 내부에서 이미 쓰이고 있는 브랜드 이름은 **책갈피**(`AppShell.tsx`의 library rail brand, `library-rail-brand-name`)이며, `frontend/app/layout.tsx`의 `JP Vocab Reader`는 지금까지 SEO title에 노출되지 않던 내부 프로젝트명이었습니다. 이번 정리에서 SEO title에도 "책갈피"를 실제 브랜드명으로 노출하도록 맞췄습니다.

## 2. 대상 사용자

- 일본어 원서·웹소설을 원문으로 읽고 싶은 한국어 사용자
- 이미 어느 정도 문법/기초 어휘는 아는데, "모르는 단어"가 읽기를 자꾸 끊어놓는 사람
- Anki 같은 범용 암기 카드 도구를 써봤지만, "단어장부터 만들고 외우는" 순서가 부담스러웠던 사람
- JLPT 시험 자체가 목적이 아니라, 원문을 즐겁게 읽는 것이 목적인 사람 (JLPT 학습자도 보조적으로 포함되지만 메인 타겟은 아님)

## 3. 사용자가 겪는 문제

1. 일본어 원문을 읽다가 모르는 단어가 나올 때마다 흐름이 끊긴다.
2. 사전에서 뜻을 찾아도 그 자리에서 휘발되고, 나중에 다시 안 외워진다.
3. 범용 암기 도구(Anki 등)는 단어장을 미리 만들어야 하는데, 그 준비 과정 자체가 진입장벽이다.
4. 어떤 단어를 외워야 할지 스스로 고르기 어렵다 — 이미 아는 단어까지 카드로 만들게 된다.

## 4. 서비스가 해결하는 방식

- 원문을 붙여넣으면 자동으로 단어를 추출하고, 한국어 뜻과 읽기를 바로 보여준다.
- 사용자는 "이미 아는 단어"를 걸러내고 "모르는 단어"만 담으면 된다 — 단어장을 미리 준비할 필요가 없다.
- 담은 단어는 SRS(간격 반복)로 자동 예약되어, 다시 읽기 전에 짧게 복습한다.
- 복습을 마치면 다시 원문을 읽으러 돌아간다 — 읽기가 계속 중심에 있다.

## 5. 핵심 루프

```
원문 입력 → 자동 단어 추출 → 뜻/읽기 확인 → 모르는 단어 저장 → SRS 복습 → 다시 원문 읽기
```

모든 카피(랜딩, 히어로, 온보딩)는 이 루프의 어느 지점을 설명하는지 스스로 확인할 수 있어야 합니다. 이 루프 밖의 이야기(예: 고급 사전 기능, 통계 상세)는 첫 화면 카피의 우선순위가 아닙니다.

## 6. 주요 차별점

| 차별점 | 한 줄 표현 |
|---|---|
| 단어장이 아니라 읽기가 먼저 | "단어장부터 만드는 서비스가 아니라, 읽다 보면 단어장이 쌓이는 서비스" |
| 선택적 저장 | "모르는 단어만 골라 저장 — 이미 아는 단어는 카드로 만들지 않아요" |
| 한국어 뜻 중심 | "일본어 원문 옆에 바로 한국어 뜻" |
| 추천 어휘는 보조 | "레벨별 추천 어휘 덱은 참고용 — 핵심은 내가 직접 읽은 원문에서 쌓은 단어장" |
| 저장 범위 최소화 | "원문 전체가 아니라 내가 고른 단어와 짧은 예문만 저장돼요" |
| 읽기 흐름 중심 | "Anki 같은 범용 암기 도구가 아니라, 읽는 흐름을 안 끊는 게 목표" |

## 7. 쓰지 말아야 할 표현

- **"공식 JLPT"** 계열 — 공식 인증/제휴가 아닙니다. 항상 "JLPT 추천 어휘", "레벨별 추천 어휘"로만 표현합니다.
- **"복사된 단어"** — 구독형 공유 덱(lexeme 기반)은 실제로 개인 단어장에 복사되지 않습니다. "가져온 덱", "학습 목록에 추가"로 표현합니다.
- **"원문 전체를 저장한다/공유한다"**처럼 읽히는 표현 — 실제로는 저장하지 않습니다. 항상 부정형("원문 전체는 저장하지 않아요")으로만 언급합니다.
- **"AI가 다 해준다"** 식 과장 — 형태소 분석·사전 매칭 기반 기능을 AI 만능형으로 부풀리지 않습니다.
- **키워드 나열식 문구** — "일본어, 단어장, SRS, 웹소설, 원서, 암기, 복습, ..." 식으로 명사만 늘어놓지 않습니다. 항상 자연스러운 한 문장으로 씁니다.
- **베타 단계에 안 맞는 확신형 표현** ("모든 사용자가", "1위", "완벽한") — 서비스는 아직 베타 단계입니다.

## 8. 추천 어휘 덱 설명 원칙

- 추천 어휘 덱(주로 JLPT 레벨별)은 **핵심 기능이 아니라 보조 콘텐츠**입니다 — 서비스의 본질은 "내가 읽은 원문에서 만든 단어장"이고, 추천 덱은 시작할 원문이 아직 없는 사용자를 위한 보조 수단입니다.
- 명칭은 항상 **"JLPT 추천 어휘"** 또는 **"레벨별 추천 어휘"**로 표현하고 "공식", "인증" 같은 단어를 붙이지 않습니다.
- 설명 문구에는 "학습 참고용 비공식 목록"이라는 사실을 짧게라도 포함합니다 (이미 `SharedDeckSection.tsx`에 있는 디스클레이머 문구가 기준).
- 추천 덱을 가져오는 행위는 "가져오기"/"학습 목록에 추가"로 표현하고, 개인 단어장과 구독형 공유 덱의 저장 방식 차이(복사 vs 구독)를 사용자에게 굳이 노출하지 않습니다 — 사용자 입장에서는 "학습 목록에 추가된다"는 결과만 알면 충분합니다.

현재 `HomeDashboard.tsx` 히어로의 길이/톤(두 줄 제목 + 한 줄 부제, 해요체, 캐주얼)에 맞춘 후보입니다.

**후보 A (채택)**
- 제목: "오늘 읽을 일본어 원문을 / 펼쳐볼까요?"
- 부제: "모르는 단어를 눌러두면, 읽으면서 단어장이 자연스럽게 쌓여요."

**후보 B**
- 제목: "일본어 원문을 읽으며 / 단어장을 만들어요"
- 부제: "모르는 단어만 저장하고, 짧게 복습한 다음 다시 읽어요."

**후보 C**
- 제목: "읽다가 모르는 단어만 / 책갈피처럼 담아요"
- 부제: "일본어 원문 읽기와 단어장, 복습을 한 흐름으로 이어드려요."

- "일본어 원문을 붙여넣고 모르는 단어를 눌러보세요." (현재 읽기 탭 empty-state와 톤 일치, 채택 유지)
